# Invoice Processing Agent

An AI agent that processes invoice PDFs using the [Anthropic tool-use API](https://docs.anthropic.com/en/docs/tool-use). Claude autonomously decides which tools to call and in what order — no hardcoded workflow.

## How it works

The agent receives raw invoice text and has access to three tools:

| Tool | What it does |
|---|---|
| `extract_invoice_data(pdf_text)` | Parses invoice number, PO number, total due, and due date from raw text |
| `lookup_po(po_number)` | Checks a PO database and returns the vendor name and approved amount |
| `flag_exception(reason)` | Flags the invoice for human review with a written reason |

Claude decides which tools to call based on what it observes. A clean invoice flows: extract → lookup → approve. A problematic invoice flows: extract → lookup → flag. A missing PO skips lookup entirely and goes straight to flag. The agent loop runs until Claude returns `stop_reason == "end_turn"` with no tool calls.

## Setup

**Requirements:** Python 3.10+

```bash
pip install anthropic pypdf reportlab
```

Set your Anthropic API key as an environment variable:

```bash
# Linux / macOS
export ANTHROPIC_API_KEY=sk-ant-...

# Windows (PowerShell — persists for this session only)
$env:ANTHROPIC_API_KEY = [System.Environment]::GetEnvironmentVariable("ANTHROPIC_API_KEY", "User")
```

## Usage

**Run the agent on a single invoice:**

```bash
python agent.py mock_invoice.pdf
python agent.py test_invoices/invoice_over.pdf
```

**Generate the four test invoices:**

```bash
python create_test_invoices.py
```

**Run all tests and print a summary table:**

```bash
python run_tests.py
```

Test invoices are generated by `create_test_invoices.py` using `reportlab`. Each PDF has mathematically exact totals (subtotal × 1.18 = TOTAL DUE, no rounding hacks).

## PO database

`agent.py` contains a mock `PO_DATABASE` dict. To add a new PO:

```python
PO_DATABASE = {
    "PO-1001": {"vendor": "Acme Supplies",    "approved_amount": 50000, "currency": "INR"},
    "PO-1002": {"vendor": "Bright Logistics", "approved_amount": 30000, "currency": "INR"},
    "PO-XXXX": {"vendor": "Your Vendor",      "approved_amount": 99999, "currency": "INR"},
}
```

## Project structure

```
agent.py                  # Agent loop + tool definitions + PO database

run_tests.py              # Batch runner — prints pass/fail summary table
mock_invoice.pdf          # Original sample invoice (Acme Supplies, INV-2026-0892)
test_invoices/            # Generated test PDFs (gitignore or commit as fixtures)
```
